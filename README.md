# ReportBridge

Collezione di scaffold del **Report Bridge (RB)** per diverse tipologie di aziende.

Parte del progetto editoriale **Receive/Declutter Methodology book**.

## Cos'è

ReportBridge e un repository di strutture pronte all'uso per organizzare la conoscenza tecnica e operativa di un team software in modo semplice, versionato e collaborativo.

## Obiettivo

- Ridurre la dispersione delle informazioni
- Aumentare continuita e condivisione della conoscenza
- Offrire basi adattabili a contesti aziendali diversi

## Scelta Rapida Scaffold

| Contesto | Scaffold consigliato | Livello iniziale |
|---|---|---|
| SaaS di prodotto | saas-product | standard |
| Software house / agenzia | software-house | lite |
| Team enterprise interno | enterprise-internal | standard |
| Startup early-stage | startup-early-stage | lite |

## Cosa Trovi Nel Repo

- Scaffold per 4 tipologie di azienda, ciascuno con 3 livelli: lite, standard, advanced
- Template pronti: ADR, bug log, runbook incidenti, onboarding, convenzioni di codice, glossario e mission di progetto
- Esempi compilati per partire subito
- Guide rapide: scelta scaffold, quickstart 30 minuti, tassonomia tag, metriche adozione

1. Scegli scaffold e livello in [guides/choose-scaffold.md](guides/choose-scaffold.md)

Il repository non impone una struttura rigida: offre una base concreta da personalizzare in base al contesto.
