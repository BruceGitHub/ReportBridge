# Tag Taxonomy

Use simple, stable tags. Avoid creating near-duplicates.

## Core Dimensions

- area: backend, frontend, infra, data, security
- type: adr, bug, runbook, procedure, standard
- priority: p0, p1, p2, p3
- status: draft, active, deprecated, resolved
- risk: low, medium, high
- owner: team or person identifier

## Naming Rules

- lowercase only
- use hyphen for multi-word tags
- one canonical tag per concept

## Examples

- #area-backend #type-bug #priority-p1 #status-active #risk-medium #owner-platform
- #area-infra #type-runbook #priority-p0 #status-active #risk-high #owner-sre
