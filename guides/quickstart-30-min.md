# Quickstart in 30 Minutes

## Goal

Start a usable Report Bridge with minimum friction.

## Checklist

1. Pick scaffold and level from choose-scaffold.md.
2. Create the main RB folder in your target repository.
3. Copy core templates:
   - adr_template.md
   - bug_template.md
   - runbook_incident_template.md
   - onboarding_template.md
   - coding_conventions_template.md
   - adoption_metrics_template.md
   - tech_debt_template.md
   - improve_template.md
   - known_errors_template.md
   - customer_procedure_template.md
   - faq_template.md
   - coding_template.md
   - testing_strategy_template.md
   - settings_explained_template.md
   - naming_conventions_template.md
   - coding_standard_template.md
   - dsl_code_level_template.md
   - glossary_template.md
   - project_mission_template.md
4. Fill one real ADR and one real bug entry.
5. Define owners for RB update and review cadence.
6. Agree on tag taxonomy and naming.
7. Share RB location with the whole team.

## Done Criteria

Your RB is active when:

- at least one ADR is written
- at least one operational or bug document is written
- team members know where to write and where to read
